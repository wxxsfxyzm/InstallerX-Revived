# 用户配置文件

在本页面，您可以新增配置，配置可以关联特定应用，以在应用发起安装时自动生效

## 作用域

定义配置生效的应用列表，核心功能，决定您的配置向特定应用生效

## 授权器

决定 InstallerX-Revived 进行安装操作时使用何种方式获取权限进行安装

* **Root:** 适用于已安装 Magisk/KernelSU/APatch 的设备。
* **Shizuku:** 推荐大多数用户使用。对于 Root 用户，也建议优先使用 Shizuku，因为通常情况下它的性能更好。
* **Dhizuku:** 设备所有者模式（功能受限但对一些设备特别有用）。
* **无:** 调用系统原生包管理器进行确认安装（仅在支持的设备上显示）。
* **自定义:** 自定义 InstallerX-Revived 通过执行自定义命令获取权限

## 安装方式

决定 InstalllerX-Revived 在安装过程中使用何种界面向用户展现

* **对话框:** 弹出包含应用详情的对话框，需手动点击“安装”。
* **对话框自动:** 弹出对话框，但自动进行安装。
* **通知栏:** 在后台安装，通过标准通知栏进度条显示进度，需要手动确认。
* **通知栏自动:** 同上，但无需用户手动确认，进一步减少交互干扰。
* **忽略:** 忽略命中本配置的应用的安装请求。

## 安装器设置

决定 InstallerX-Revived 要执行的自定义行为

::: info 小米 / HyperOS 用户注意
在 MIUI/HyperOS 上更新或安装系统应用时，系统要求“安装者”必须也是系统应用。默认情况下，InstallerX Revived 会自动将 **安装来源** 声明为 `com.miui.packageinstaller` 以确保兼容性。除非您是高级用户，否则无需修改。
:::

* **设定用户:** 指定命中本配置的应用要对何用户安装应用，而非向当前用户安装。
* **自定义安装原因:** 向 Android 操作系统提供自定义的安装原因，以让系统进行进一步的操作。
* **自定义软件包来源:** 向 Android 操作系统提供自定义的软件包来源指示，以让系统进行进一步的操作。
* **声明安装者:** 向 Android 操作系统提供自定义的安装来源指示，以让系统进行进一步的操作。
* **手动 Dex 优化:** 在安装后触发系统 dex2oat 编译。 [了解更多](https://source.android.com/docs/core/runtime/configure?hl=zh-cn)
* **自动删除:** 在安装完成后自动删除安装包。
* **显示 SDK 信息:** 在安装界面展示目标 Android API 版本和最低 Android API 版本。[了解更多](https://developer.android.com/guide/topics/manifest/uses-sdk-element?hl=zh-cn#ApiLevels)
* **显示安装包大小:** 在安装界面展示安装包大小信息

## 安装选项

决定 InstallerX-Revived 向系统提供的安装请求标志

::: warning 允许降级安装
在 Android 15 以上版本，此功能需要 Root 权限或作为系统安装器运行
:::

::: warning 授权所有请求权限
此功能只适用于运行时权限
:::

* **为所有用户安装:** 为系统上的所有用户安装软件包，而不仅仅是指定或默认用户。
* **允许测试包:** 允许安装带 TestOnly 标签的安装包。
* **允许降级安装:** 允许降级安装低版本的安装包。
* **绕过低目标 SDK 限制:** 允许安装针对旧版本 Android 的应用。
* **授权所有请求权限:** 自动授予包请求的所有权限。

## 拆分安装包

决定 InstallerX-Revived 对于分包应用的安装行为

* **默认选中所有分包:** 安装时默认选中所有分包，而不是选中最合适的。
